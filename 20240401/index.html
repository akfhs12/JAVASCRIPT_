반응형웹(response web)이란?

하나의 html 문서를 제작하여 pc, tablet, mobile 화면에서 최적화된 콘텐츠를 보여주기 위한 기술

2. 뷰포트속성(viewport)

사용자의 디바이스 화면크기(pc, tablet, mobile)에 맞추어 웹문서를 100%로 보여줄 수 있는 속성으로 pc보다 태블릿이나 모바일 사용자를 위해 필요

반응형 웹 기술 3가지

  1. 가변단위 - em, rem, %, vh, vw...
  2. 뷰포트속성 - meta name='viewport'
  3. 미디어쿼리 - 해상도별 css 적용하기

반응형 웹 참고 - 디비컷, https://mediaqueri.es

웹문서의 상단 head에 meta="viewport" content="width=deviec-width, initial-scale=1"내용을 삽입

  • name="viewport" : pc, tablet, mobile 디바이스 화면에 맞게 설정
  • width=device-width : 콘텐츠 내용을 pc, tablet, mobile 화면 가로너비와 일치시킴
  • initial-scale=1 : 디바이스 화면에 1:1 비율로 콘텐츠를 보이게 함. 숫자 2를 입력하면 화면 두 배로 커짐

3. 미디어쿼리(mediaquery) 속성

브라우저의 가로너비(해상도)에 따라 또는 사용자의 디스플레이 가로 해상도에 따라 css서식을 적용하기 위한 기술로 반응형 웹페이지 제작시 필수 속성

해상도 참고사이트: https://screensiz.es/phone, http://iosres/com

미디어쿼리 작성시 각각의 디바이스별 해상도에 대한 이해가 필요한데 모바일 영역은 0 ~ 767px, 태블릿 영역은 768px ~ 1024px, PC영역은 1025px 이상의 영역 설정

각 디바이스별 해상도 기준을 정하는 것을 'Break Point(중단점)'라고 한다.

        미디어쿼리 작성법
        @media screen and (디바이스별 가로 해상도) {
          조건에 맞는 해상도일 때 적용할 css서식;
        }
